**Add to storage/index.ts:**
- `export async function getAllSitePreferences(): Promise<SitePreference[]>` — returns `db.sitePreferences.toArray()`
- `export async function deleteSitePreference(domain: string): Promise<void>` — removes site preference
- `export async function deleteAllSitePreferences(): Promise<void>` — clears all site preferences
- `export async function getStorageStats(): Promise<{notes: number, sitePreferences: number, analytics: number}>` — returns count per table using `db.table.count()`

**HotkeysSettings.tsx:**
- Call `chrome.commands.getAll()` on mount to get registered commands
- Render a table: Action Name column | Keyboard Shortcut column
- Action name = `cmd.description` (fallback to `cmd.name`)
- Shortcut: if `cmd.shortcut` exists show in `<kbd>` styled element, otherwise show muted gray "Not set" badge
- Below table: instruction paragraph "To change shortcuts, open Chrome's extension shortcuts page (chrome://extensions/shortcuts)"
- Use `chrome.tabs.create({ url: 'chrome://extensions/shortcuts' })` if clicking the link programmatically is possible, otherwise just show the URL as text (chrome:// URLs can't be linked from extension pages — just show the instruction)

**PerSiteSettings.tsx:**
- On mount, call `getAllSitePreferences()` to load all site overrides
- Each site preference renders as a card showing:
  - Domain name (the `id`) as card header
  - List of non-null overrides: "Font: OpenDyslexic", "Line Spacing: 1.8", etc.
  - Click card to expand — shows full edit form with Toggle/Slider/Select for visual settings (fontEnabled, fontFamily, lineSpacing, letterSpacing, theme)
  - Each override field has a "Reset" button to set it back to null (use global)
  - Card footer: "Remove all overrides for this site" button with confirmation
- Expanded edit state: local useState per card
- Empty state when no site preferences exist: "No site-specific preferences. You can add per-site overrides from the extension popup."
- Only show visual settings per CONTEXT.md (fontEnabled, fontFamily, lineSpacing, letterSpacing, theme) — NOT companion behavior

**PrivacySettings.tsx:**
- Storage Usage section:
  - Call `getStorageStats()` on mount
  - Show item counts per category: "Notes: {n} items", "Site Preferences: {n}", "Analytics entries: {n}"
  - Use plain text format (Claude's discretion per CONTEXT.md)
- Export section:
  - Single "Export My Data" button
  - On click: gather all data from Dexie tables (settings, notes metadata without audioBlob, sitePreferences, analytics)
  - Strip audioBlob from notes (too large for JSON) — map to exclude it
  - Create JSON blob, trigger download via createObjectURL + anchor click
  - Filename: `dyslexia-tool-export-{YYYY-MM-DD}.json`
- Delete section (nuclear):
  - "Delete All Data" button (red, destructive styling)
  - On click: show confirmation panel with text input
  - User must type "DELETE" (case-insensitive comparison: `input.toUpperCase() === 'DELETE'`)
  - Show live matching progress: as user types, show how close they are (e.g., "DEL..." → "DELETE ✓")
  - On confirm match: call `db.settings.clear()`, `db.notes.clear()`, `db.sitePreferences.clear()`, `db.analytics.clear()`, then `initializeStorage()`, then `window.location.reload()`
  - Auto-focus the input when confirmation panel appears
