**File 1: Create `popup/Onboarding.tsx`**

Build a 3-step onboarding component per the locked decisions from CONTEXT.md:

```typescript
interface OnboardingProps {
  onComplete: () => void

```

**Design:**
- Full-width within the popup (400px width, same as main UI)
- Step indicator at top (3 dots, current step highlighted in blue #3B82F6)
- Content area with icon/illustration, heading, description per step
- "Next" button (blue), "Skip" link (gray), final step has "Try it now" button (green #10B981)
- Warm, supportive tone throughout — "Let's make reading easier for you." style language

**Step 1 — "Welcome to Dyslexia Tool"**
- Heading: "Let's make reading easier for you."
- Description: Brief, warm explanation of what the extension does — applies dyslexia-friendly fonts, offers spelling help while you type, reads text aloud.
- Visual: Large, friendly icon (use emoji 💙 or Unicode book 📖 — no external assets needed)

**Step 2 — "Your Reading Companion"**
- Heading: "A friend that notices when you need help"
- Description: Explain the companion — it watches for typing struggles and offers spelling suggestions. No judgment, just gentle help. Shows a visual example of what the suggestion popup looks like.
- Include a small mock/preview of the suggestion UI (e.g., a styled div showing "Suggestions for 'becuase': because" with the blue styling)

**Step 3 — "Make It Yours"**
- Heading: "Customize your experience"
- Description: Mention settings page where they can adjust fonts, companion sensitivity, and more.
- Tell them they can always find settings by right-clicking the extension icon → Options.
- Final CTA: "Try it now" button (green, prominent)

**"Try it now" CTA behavior:**
Open a new tab to `https://en.wikipedia.org/wiki/Dyslexia` — this is a stable URL with real, meaningful text about dyslexia. The content script runs on all pages, so the companion will be active. Call `chrome.tabs.create({ url: 'https://en.wikipedia.org/wiki/Dyslexia' })`.

**State management:**
- Local `useState` for current step (1, 2, 3)
- `onComplete` callback when finished (sets chrome.storage.local flag + closes onboarding)
- "Skip" link on any step also calls `onComplete`

**Styling:** Use inline styles matching the existing popup style (same font sizes, colors, spacing). The popup currently uses inline styles (not Tailwind). Keep consistent.

**File 2: Update `popup/App.tsx`**

Add first-install detection and conditional onboarding rendering:

1. Add state: `const [showOnboarding, setShowOnboarding] = useState(false)`
2. In the existing `useEffect` that loads data (line ~57-73), add:
   ```typescript
   chrome.storage.local.get('onboardingComplete', (result) => {
     setShowOnboarding(!result.onboardingComplete)
   })
   ```
3. After the loading/error checks, add conditional:
   ```typescript
   if (showOnboarding) {
     return <Onboarding onComplete={async () => {
       await chrome.storage.local.set({ onboardingComplete: true })
       setShowOnboarding(false)
     }} />
   }
   ```
4. Import `Onboarding` from `./Onboarding`
